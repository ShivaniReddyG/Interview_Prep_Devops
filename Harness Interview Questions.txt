# Top 100 Harness Interview Questions with Answers

# Harness Fundamentals

### 1. What is Harness?

Harness is a modern CI/CD and DevOps platform used to automate software build, test, deployment, verification, and rollback processes.

---

### 2. What are the key features of Harness?

* CI/CD automation
* Continuous Verification
* GitOps
* Feature Flags
* Cloud Cost Management
* Security Testing
* Automated Rollback

---

### 3. What problems does Harness solve?

* Manual deployment errors
* Slow release cycles
* Lack of rollback automation
* Deployment visibility issues
* CI/CD complexity

---

### 4. Explain the architecture of Harness.

Main components:

* Harness Manager (SaaS platform)
* Delegates
* Connectors
* Pipelines
* Verification Engine

---

### 5. What are the modules available in Harness?

* CI
* CD
* STO
* SRM
* CCM
* Feature Flags
* Chaos Engineering

---

### 6. Difference between Harness CI and Harness CD?

| Harness CI                  | Harness CD            |
| --------------------------- | --------------------- |
| Build and test applications | Deploy applications   |
| Generates artifacts         | Releases artifacts    |
| Focuses on integration      | Focuses on deployment |

---

### 7. What is a Harness pipeline?

A pipeline is an automated workflow consisting of stages and steps.

---

### 8. What are stages in Harness?

Stages are logical deployment units like:

* Build
* Deploy
* Approval
* Security Scan

---

### 9. What are steps in Harness?

Steps are individual tasks executed inside stages.

Example:

* Run script
* Build Docker image
* Deploy manifest

---

### 10. What are connectors in Harness?

Connectors securely integrate external systems:

* GitHub
* Kubernetes
* Docker Hub
* AWS
* Terraform

---

# Delegates & Infrastructure

### 11. What is a Harness Delegate?

A lightweight agent installed in infrastructure to execute deployment tasks securely.

---

### 12. Why is Delegate required?

* Access private infrastructure
* Execute deployments
* Fetch secrets
* Run scripts

---

### 13. Types of Delegates in Harness?

* Kubernetes Delegate
* Docker Delegate
* Helm Delegate
* ECS Delegate

---

### 14. How do you install a Delegate?

Using:

* Kubernetes YAML
* Helm chart
* Docker container

---

### 15. How does Delegate communicate with Harness Manager?

Using outbound HTTPS connection over port 443.

---

### 16. What ports are required for Delegate communication?

Typically:

* 443 HTTPS
* Cluster-specific ports if needed

---

### 17. How do you troubleshoot disconnected Delegates?

Check:

* Network connectivity
* Proxy/firewall
* Delegate pod status
* Tokens
* Logs

---

### 18. How do you scale Delegates?

* Increase replicas
* Use autoscaling
* Deploy multiple Delegates

---

### 19. Explain Delegate selectors.

Selectors map pipelines to specific Delegates.

---

### 20. What happens if a Delegate fails during deployment?

Another eligible Delegate can pick up the task if redundancy exists.

---

### 23. What deployment strategies are supported?

* Rolling

---

### 24. Explain Rolling deployment.

Pods are updated gradually without downtime.

---

### 25. Explain Blue-Green deployment.

Two environments exist:

* Blue → Current
* Green → New version

---

### 26. Explain Canary deployment.

Deploy to a small subset of users before full rollout.

### 27. Difference between Blue-Green and Canary deployment?

| Blue-Green              | Canary          |
| ----------------------- | --------------- |
| Full environment switch | Gradual rollout |
| Faster rollback         | Better testing  |
