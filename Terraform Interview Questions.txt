# Terraform Interview Questions with Answers

## 1. What is Terraform?

Terraform is an Infrastructure as Code (IaC) tool by HashiCorp used to provision and manage infrastructure using configuration files.

---

## 2. What is Infrastructure as Code (IaC)?

IaC is the process of managing infrastructure using code instead of manual configuration.

Benefits:

* automation
* consistency
* version control
* repeatability

---

## 3. Why is Terraform used in DevOps?

Terraform helps:

* automate infrastructure
* maintain consistency
* support CI/CD
* enable multi-cloud deployments

---

## 4. Advantages of Terraform

* Open source
* Multi-cloud support
* Declarative syntax
* Reusable modules
* State management

---

## 5. Terraform vs Ansible

| Terraform              | Ansible                  |
| ---------------------- | ------------------------ |
| Provisioning tool      | Configuration management |
| Declarative            | Procedural               |
| Creates infrastructure | Configures servers       |

Ansible is often used together with Terraform.

---

## 6. Terraform vs CloudFormation

| Terraform     | CloudFormation |
| ------------- | -------------- |
| Multi-cloud   | AWS only       |
| HCL syntax    | JSON/YAML      |
| More flexible | AWS-native     |

---

# State Management

## 31. What is state file?

`terraform.tfstate` stores infrastructure mappings.

---

## 32. Why is state important?

Terraform uses state to determine changes required.

---

## 33. What if state file is deleted?

Terraform loses tracking information.

Recovery:

* restore backup
* import resources again

# Troubleshooting & Scenario-Based Questions

## 91. Terraform apply failed. What will you do?

Steps:

1. Check error logs
2. Run:

```bash id="qj4n8w"
terraform plan
```

3. Verify provider credentials
4. Check resource conflicts
5. Retry after fixing issues

---

## 92. State file corrupted. How to recover?

* Restore backup state
* Use remote backend versioning
* Re-import resources if needed

# Important Scenario-Based Answers

## Secret accidentally committed

Steps:

1. Remove secret from repository
2. Rotate credentials
3. Rewrite Git history if needed
4. Move secrets to Vault

---

## Infrastructure drift detected

Steps:

1. Run:

```bash id="j0nqxu"
terraform plan
```

2. Compare differences
3. Reconcile infrastructure
4. Apply updates carefully

---

## Terraform deployment failed midway

Steps:

1. Check partial resources
2. Verify state consistency
3. Run plan again
4. Import missing resources if needed

---

## Multiple engineers working on same infrastructure

Best practices:

* Remote backend
* State locking
* Separate environments
* Pull request approvals
