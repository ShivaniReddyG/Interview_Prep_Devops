# Docker Interview Questions with Answers for DevOps Engineers

## 1. What is Docker?

Docker is an open-source containerization platform used to package applications and dependencies into lightweight containers.

---

## 2. What problem does Docker solve?

Docker solves:

* “Works on my machine” issues
* Dependency conflicts
* Environment inconsistency
* Deployment portability

---

## 3. Docker vs Virtual Machines

| Docker              | Virtual Machine     |
| ------------------- | ------------------- |
| Lightweight         | Heavy               |
| Shares host kernel  | Separate OS         |
| Fast startup        | Slow startup        |
| Less resource usage | More resource usage |

---

## 4. What is containerization?

Containerization packages application code and dependencies together to run consistently across environments.

---

## 5. What is a Docker image?

A read-only template used to create containers.

Example:

```bash id="lhr1lt"
docker pull nginx
```

---

## 6. What is a Docker container?

A running instance of a Docker image.

---

## 7. Image vs Container

| Image    | Container        |
| -------- | ---------------- |
| Template | Running instance |
| Static   | Dynamic          |

---

## 8. What is Docker Engine?

Core Docker runtime consisting of:

* Docker daemon
* REST API
* Docker CLI

---

## 9. Components of Docker Architecture

* Docker Client
* Docker Daemon
* Docker Registry
* Docker Objects

---

## 10. What is Docker Hub?

Cloud registry for storing and sharing Docker images.

Alternative registries:

* Harbor
* GitLab Container Registry

---

# Docker Architecture

## 11. Docker client vs daemon

| Client            | Daemon                    |
| ----------------- | ------------------------- |
| User interface    | Background service        |
| Executes commands | Manages containers/images |

---

## 12. What is container runtime?

Software responsible for running containers.

Examples:

* containerd
* CRI-O

---

## 13. What is containerd?

A container runtime managing container lifecycle.

---

## 14. What is runc?

Low-level runtime used to create/run containers.

---

## 15. How Docker communicates with kernel?

Using:

* namespaces
* cgroups
* Linux kernel features

---

## 16. What are namespaces?

Provide isolation for:

* process IDs
* network
* filesystem
* users

---

## 17. What are cgroups?

Control resource usage:

* CPU
* memory
* disk I/O

---

## 18. What is Union File System?

Allows multiple layers to appear as single filesystem.

---

## 19. What is overlay2?

Default Docker storage driver using layered filesystem.

---

## 20. What is Docker Registry?

Storage system for Docker images.

---

# Docker Images

## 31. What is Dockerfile?

Text file containing image build instructions.

---

## 32. Explain image layers

Each Dockerfile instruction creates a layer.

Benefits:

* caching
* reuse
* smaller downloads

---

## 33. Intermediate layers

Temporary layers created during image build.

---

## 34. What is image caching?

Docker reuses unchanged layers to speed builds.

---

## 35. COPY vs ADD

| COPY             | ADD                          |
| ---------------- | ---------------------------- |
| Simple file copy | Supports URLs/tar extraction |

Prefer COPY.

---

## 36. CMD vs ENTRYPOINT

| CMD               | ENTRYPOINT      |
| ----------------- | --------------- |

---

## 38. Purpose of EXPOSE

---

# Dockerfile Best Practices

## 41. Why use Alpine images?

* Small size
* Faster downloads
* Reduced attack surface

---

## 42. What is multi-stage build?

Uses multiple build stages to reduce final image size.

Example:

```dockerfile id="j8skud"
FROM maven AS build
FROM openjdk
```

---

## 43. Why avoid running as root?

Improves security and reduces privilege escalation risks.

---

## 44. Optimize Docker build performance

* Layer caching
* `.dockerignore`
* Smaller contexts

---

## 45. What are dangling images?

Unused images without tags.

---

## 46. What is `.dockerignore`?

Excludes files from build context.

Example:

```bash id="ys9k7y"
node_modules
.git
```

---

## 47. Pass variables during build

```bash id="axifnf"

```

---

## 48. ARG vs ENV

| ARG           | ENV       |
| ------------- | --------- |

| Not persisted | Persisted |

---

## 49. What is build context?

Files accessible during Docker build.

---

## 51. Explain Docker networking

Enables communication:

* container-to-container
* container-to-host
* external access

---

## 52. Types of Docker networks

* bridge
* host
* overlay
* none
* macvlan

---

## 53. What is bridge network?

Default private network for containers.

---

## 54. What is host network?

Container shares host networking stack.

---

## 55. What is overlay network?

Used in clustering across multiple hosts.

---

## 56. What is macvlan?

Assigns MAC address to containers.

---

## 57. Container communication

Containers communicate through Docker networks and DNS.

---


# Docker Volumes

---

| ------------- | -------------- |
| Host path     | Docker-managed |
| Less portable | More portable  |

---

## 63. What are tmpfs mounts?

---

## 64. How volumes persist data?

---

## 68. Explain storage drivers

## 69. Clean unused volumes

## 70. What happens after container deletion?
